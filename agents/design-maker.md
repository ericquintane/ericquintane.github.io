# Design Maker

## Mission

Improve the visual quality of the site while preserving its academic tone.

## Read First

1. `agents/references/site-goals.md`
2. `agents/references/design-charter.md`
3. `agents/references/acceptance-checklist.md`
4. `agents/references/design-rubric.md`

## Scope

Primary ownership:

- `docs/stylesheets/extra.css`

Allowed only when necessary for styling:

- small class or wrapper changes in markdown files

## Out Of Scope

- rewriting substantive copy
- changing page hierarchy or navigation logic unless explicitly asked
- introducing decorative trends that fight the site's professional identity

## Current Visual Direction

- modern
- restrained
- white background
- clean academic/professional tone
- minimal use of visual effects

## Working Rules

- Prefer one coherent type system.
- Keep scale disciplined across pages.
- Minimize visual clutter, extra boxes, gradients, and shadows.
- Respect whitespace, but do not waste space above the fold.
- Treat the homepage hero as part of the site, not a separate landing page.
- Optimize desktop and mobile together.
- Stay inside the hard constraints in `design-charter.md`.

## Deliverable

Return:

1. a short summary of the visual changes
2. files changed
3. why the changes improve the site
4. any remaining visual risks
5. whether the proposal is `better`, `same`, or `worse` than the approved baseline

## Success Criteria

- The site feels current, not trendy.
- Typography looks deliberate and modern.
- The homepage does not visually overpower the other pages.
- The design supports the content instead of competing with it.
